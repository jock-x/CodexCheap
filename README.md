# CodexCheap

CodexCheap 是一个用于比较 Codex/Codex 类中转站价格的比价系统，重点把不同网站的按量充值、倍率、套餐时限和套餐额度折算到统一口径，帮助用户快速看出“按量使用”和“套餐使用”哪一种更便宜。

系统包含公开比价页和后台维护页。游客可以直接查看价格比较；后台只保留一个管理员账号，用于维护中转站、按量充值套餐、倍率号池、套餐和套餐额度。

## 项目定位

CodexCheap 的目标不是做完整导航站，而是做一个可维护、可计算、可排序的价格比较工具。

它主要解决这些问题：

- 不同中转站的充值金额、兑换额度、倍率、套餐限制不一致，直接对比很困难。
- 按量使用和套餐使用的便宜程度不一样，需要分开比较。
- 套餐可能有日限额、周限额、月限额或总额度，需要统一折算为 `元 / $1 / 天`。
- 按量倍率可能按号池分组区分，例如 `Pro`、`Plus`、`Team`、`未知`。

## 当前覆盖范围

当前数据仅覆盖作者已知并手动录入的网站，不保证覆盖全网，也不保证价格实时准确。

中转站价格、套餐规则、倍率、额度、有效期可能会随时变化。使用前请以对应网站的实际页面为准。

## 功能特性

- 公开比价页：无需登录即可查看按量比较和套餐比较。
- 按量比较：按中转站聚合，支持号池分组筛选，按最低 `元 / $1` 排序。
- 套餐比较：按套餐日均成本比较，支持查看套餐额度明细。
- 后台维护：维护中转站、按量充值套餐、倍率号池、套餐和套餐额度。
- 单管理员登录：仅后台需要登录，游客页面不需要账号。
- MySQL 初始化脚本：数据库结构和示例数据放在 `SQL/` 目录；管理员账号按部署环境单独初始化。

## 价格计算方式

### 按量使用

按量充值会录入人民币充值金额、兑换美元额度和倍率。

```text
有效额度 = 兑换美元 / 倍率
元 / $1 = 充值人民币 / 有效额度
```

示例：`3.5 元换 3.5 美元，倍率 1x`，有效额度为 `$3.5`，成本为 `1.0000 元 / $1`。

### 套餐使用

套餐会录入套餐价格、套餐时限、倍率和额度规则。额度规则支持日限额、周限额、月限额和套餐总额度。

```text
日均价格 = 套餐价格 / 套餐时限
日均有效额度 = 折算后的日均美元额度 / 倍率
元 / $1 / 天 = 日均价格 / 日均有效额度
```

额度折算口径：

- 日限额：直接按当天额度计算。
- 周限额：额度除以 `7`。
- 月限额：额度除以 `30`。
- 套餐总额度：额度除以套餐时限天数。

## 技术栈

- 后端：`.NET 9 Web API`
- 数据库：`MySQL`
- ORM：`FreeSql`
- 认证：`JWT`
- 前端：`React + TypeScript + Ant Design`
- 构建工具：`Vite`
- 初始化脚本：`SQL/001_init_schema.sql`、`SQL/003_sample_price_data.sql`、`SQL/004_add_pool_groups_and_recharge_rates.sql`

## 快速开始

### 1. 初始化数据库

在 MySQL 中按顺序执行 `SQL/` 目录下的公开初始化脚本：

```text
SQL/001_init_schema.sql
SQL/003_sample_price_data.sql
SQL/004_add_pool_groups_and_recharge_rates.sql
```

`001` 会创建数据库和基础表，`003` 是示例价格数据，`004` 是号池分组和按量倍率明细迁移脚本。后台管理员初始化请按部署环境单独处理，不建议在公开说明中暴露默认账号或密码。

### 2. 配置后端

`CodexCheap.API/appsettings.json` 只保留可提交的占位示例。请通过环境变量提供真实 MySQL 连接串和 JWT 密钥，不要把数据库密码或密钥写入会提交的配置文件。

PowerShell 示例：

```powershell
$env:ConnectionStrings__Default="server=<db-host>;port=3306;uid=<db-user>;pwd=<db-password>;database=codexcheap;charset=utf8mb4;TreatTinyAsBoolean=true;Allow User Variables=True"
$env:Jwt__Secret="<replace-with-a-random-secret-at-least-32-characters>"
```

公开仓库中的 `SQL/002_seed_admin.example.sql` 仅作为管理员初始化模板。复制为本地的 `SQL/002_seed_admin.sql` 并替换用户名和密码哈希后再执行；真实脚本已被 `.gitignore` 忽略。

启动后端：

```bash
dotnet run --project CodexCheap.API/CodexCheap.API.csproj
```

### 3. 启动前端

```bash
cd CodexCheap.React
npm install
npm run dev
```

如需指定后端地址，可配置前端环境变量：

```text
VITE_API_BASE_URL=http://localhost:5130
```

## 后台登录

后台仅供管理员维护价格数据使用。部署时请自行初始化管理员账号，并修改 `Jwt:Secret`。

后台入口：

```text
/admin/login
```

## 数据维护说明

后台维护的数据包括：

- 中转站：名称、地址、支持类型、启用状态、备注。
- 按量充值：充值人民币、兑换美元、有效期。
- 倍率号池：每个按量充值套餐可配置 `Pro`、`Plus`、`Team`、`未知` 等号池倍率。
- 套餐：套餐名称、价格、时限、倍率、号池分组。
- 套餐额度：日限额、周限额、月限额、套餐总额度。

公开页会基于后台维护的数据自动计算并排序。

## 如何建议新增中转站

欢迎补充更便宜或更有代表性的中转站，但建议提供完整可核验信息，避免只提交模糊价格。

建议提供：

- 中转站名称和官网地址。
- 支持类型：按量、套餐或按量+套餐。
- 按量充值规则：人民币金额、兑换美元、倍率、号池分组、有效期。
- 套餐规则：套餐价格、时限、倍率、号池分组。
- 套餐额度：日限额、周限额、月限额或套餐总额度。
- 信息来源：截图、页面链接或其它可核验依据。

README 不列具体未核验推荐站点，避免价格过期或误导。

## 后续规划

后续计划加入中转站评价体系，让比价不只看价格，也能参考实际体验。

规划方向：

- 稳定性评价：连接成功率、可用时长、异常频率。
- 响应速度：请求延迟、排队情况、峰值时段表现。
- 模型覆盖：支持的模型范围、更新速度。
- 售后体验：响应速度、问题处理质量。
- 风险提示：跑路风险、余额有效期、规则变更频率。
- 价格历史：记录价格变化，辅助判断是否长期便宜。
- 用户反馈：收集使用体验和真实评价。
